Слайд 2 / 16 — Аннотация

Аннотация

Разработана платформа для обработки экспериментальных данных в распределённой среде. Реализован полиэдральный метод поиска минимума целевой функции. Архитектура включает ASP.NET Core, React, Docker, Kubernetes. Эксперименты показали сокращение времени обработки на 33% при сохранении точности.

Реализация

  • Полиэдральный метод (Нелдера — Мида)
  • Backend: ASP.NET Core
  • Frontend: React SPA
  • Оркестрация: Kubernetes

Ключевые результаты

  • Время обработки: −33%
  • Потребление ОЗУ: 1220 → 1040 МБ
  • RSD: 1,6% (без изменений)
  • C# vs Python: 11,4× быстрее
Слайд 3 / 16 — Введение

Введение

Автоматизация обработки экспериментальных данных в распределённых исследовательских центрах требует обеспечения воспроизводимости вычислений и масштабируемости ресурсов.

Цель работы

Формализовать модель оркестрации вычислительных процессов в среде Kubernetes и подтвердить её эффективность в условиях межлабораторных сличительных испытаний.

Проблема

Разнородность методов, ручное управление, неравномерная загрузка серверов

Решение

Единая платформа с контейнеризацией и оркестрацией в Kubernetes

Слайд 4 / 16 — Формализация

Формализация задачи полиэдрального метода

Главная цель заключается в минимизации целевой функции:

S(x) = Σi=13 (1 − Ri(x) / Ai)² · 10 → min

Обозначения

  • • Ai — значения, полученные из эксперимента
  • • Ri(x) — рассчитанные значения скоростей реакций

Нормировка

Ri(x) ⇒ Ri(x) · 10−6

Вектор параметров

x = (X1, X2, X3, X4)T, где Xi ∈ [xmin; xmax], i = 1..n
Слайд 5 / 16 — Блок 3: Демонстрация

Полиэдральный метод: поиск минимума целевой функции

Задача

Минимизация суммы квадратов относительных отклонений:

S(x) = Σ (1 − R(x)/Aᵢ)²

Алгоритм

Метод деформируемого многогранника (Нелдера — Мида). Много итераций, чувствителен к задержкам.

Реализация

C# (высокая производительность) в Docker-контейнере

Слайд 6 / 16 — Операции симплекса

Операции алгоритма поиска минимума

1

Отражение

xn+3(k) = xn+2(k) + α · (xn+2(k) − xh(k))

α = 1 — коэффициент отражения

2

Растяжение

xn+4(k) = xn+2(k) + β · (xn+3(k) − xn+2(k))

β = 2,8 — коэффициент растяжения

3

Сжатие

xn+5(k) = xn+2(k) + γ · (xh(k) − xn+2(k))

γ = 0,6 — коэффициент сжатия

4

Редукция

Все векторы xi(k) уменьшаются в 2 раза отсчётом от xl(k):

xi(k+1) = (xi(k) + xl(k)) / 2

Критерий окончания

((1/(n+1)) Σi=1n+1 (S(xi(k)) − S(x̄))²) < ε

где x̄ — значение целевой функции в центре тяжести

Слайд 7 / 16 — Алгоритм

Процедура отыскания минимума S(x)

1

Отражение

Проектирование xh через центр тяжести

2

Растяжение

Увеличение шага при удачном отражении

3

Сжатие

Уменьшение шага при неудачном отражении

4

Редукция

Половинное уменьшение всех вершин
